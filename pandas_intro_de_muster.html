b) Daten holen bei lokaler Ausführung¶

Mit dem Requests-Paket knnen wir mit Webservern ber HTTP kommunizieren. Lassen Sie uns die Datei herunterladen und in das aktuelle Arbeitsverzeichnis speichern.

In [1]:
import requests

download_url = "https://raw.githubusercontent.com/fivethirtyeight/data/master/nba-elo/nbaallelo.csv"
target_csv_path = "nba_all_elo.csv"

response = requests.get(download_url)
response.raise_for_status()
with open(target_csv_path, "wb") as f:
    f.write(response.content)
print("Download ready.")

Download ready.

Datentypen mit .info() anzeigen¶

Die info()-Funktion zeigt fr alle Spalten des DataFrame die verwendeten Datentypen an. Hier hat pandas beim Import der CSV-Datei zu Beginn alle Spalten analysiert un einen mehr oder weniger geeigneten Datentyp ausgewhlt.

  • int64 ist ein ganzzahliger numerischer 64-bit Datentyp
  • float64ist eine 64-bit Gleitpunktzahl
  • object ist der allen Datentypen zugrundeliegende Basisdatentyp in Pythons Datentypenhierarchie.

Man erkennt dass fr alle numerischen Typen abhngig vom Vorhandenseins eines Kommas (oder Punkts) dezidiert entweder int64 oder float64 festgelegt, fr alle anderen Spalten jedoch der sehr allgemeine Datentyp object. Folgen und Manahmen dazu lernen Sie im weiteren Verlauf kennen.

(Mini-)Statistik mit .describe()¶

  • count: Anzahl der Zeilen, die gefllt sind (also nicht NaN)
  • mean: Arithmetisches Mittel
  • std: Standardabweichung, ugs. "Durchschnittliche Entfernung der Werte vom Mittel"

Pandas Datenstrukturen kennenlernen¶

Series: Die "Basis"-Datenstruktur in Pandas¶

Man erzeugt einen neue Series mit dem Series()-Befehl. Hier knnen z.B. Python Listen als Argument bergeben werden. Im Beispiel unten wird eine Reihe von Umstzen erzeugt.

In [20]:
revenues = pd.Series([5555, 7000, 1980])
revenues
# Expected:
# 0    5555
# 1    7000
# 2    1980
# dtype: int64
Out[20]:
0    5555
1    7000
2    1980
dtype: int64

Im Ergebnis sind zwei Spalten zu erkennen: die eigentlichen Umstze und eine Nummerierung fr jede Zeile. Zugriff auf die Werte dieser series erhlt man mit revenue.values. Das Ergebnis ist ein Python Array.

In [21]:
revenues.values
# Expected:
# array([5555, 7000, 1980])
Out[21]:
array([5555, 7000, 1980], dtype=int64)
In [22]:
type(revenues.values)
# Expected:
# <class 'numpy.ndarray'>
Out[22]:
numpy.ndarray

Die Nummerierung der Stze ist der Index ber den auf die Werte zugegriffen werden kann. Pandas RangeIndex

In [23]:

# Expected:
# RangeIndex(start=0, stop=3, step=1)
Out[23]:
RangeIndex(start=0, stop=3, step=1)

DataFrame: Pandas' beliebteste Datenstruktur¶

DataFrames sind mehrdimensionale Datenstrukturen die aus mehreren Datenreihen (-->Series) mit entsprechenden Labels (also Spaltenberschriften) und einem Index bestehen. In unserem Beispiel verbinden wir die beiden obigen Series Objekte zu einem gemeinsamen DataFrame. Der Inder wird ber das verschmelzen der beiden Indizes der ursprnglichen Series gebildet. Liegt zu einem der Indizes nicht in jeder Series ein Wert vor, so wird mit NaN aufgefllt.

In [30]:
city_data = pd.DataFrame({
    "revenue": city_revenues,
    "employee_count": city_employee_count
})
city_data
# Expected:
#               revenue         employee_count
# Amsterdam     4200            5.0
# Tokyo         6500            8.0
# Toronto       8000            NaN
Out[30]:
revenue employee_count
Amsterdam 4200 5.0
Tokyo 6500 8.0
Toronto 8000 NaN
In [31]:
city_data.index
# Expected:
# Index(['Amsterdam', 'Tokyo', 'Toronto'], dtype='object')
Out[31]:
Index(['Amsterdam', 'Tokyo', 'Toronto'], dtype='object')

Die Werte sind nun keine eindimensionalen Arrays mehr sondern, wie erwartet, zweidimensional.

In [32]:
city_data.values
# Expected:
# array([[4.2e+03, 5.0e+00],
#        [6.5e+03, 8.0e+00],
#        [8.0e+03,     nan]])
Out[32]:
array([[4.2e+03, 5.0e+00],
       [6.5e+03, 8.0e+00],
       [8.0e+03,     nan]])

Lassen wir uns die "Achsen" dieser zweidimensionalen Datenstruktur mit city_data.axes ausgeben, so sieht man, dass auch fr die Spaltenberschriften, also fr unser zwei Variablen (Umsatz und # Mitarbeiter), ein Index gebildet wurde. Diese beiden Indizes sind ihrerseits in einem Array verpackt (man bemerke die eckigen Klammern ganz auen) und so kann wie gewohnt mit [0] und [1] ber die Position im Array auf die einzelnen Objekte zugegriffen werden.

Out[35]:
Index(['revenue', 'employee_count'], dtype='object')

Achtung

Es gibt auch Unterschiede wie diese beiden Indizes auf DataFrame ebene angesprochen werden.

Die Funktion keys() liefert den "Spaltenindex", das Attribut .index liefert hingegen den "Zeilenindex". Wird mit dem in Operator direkt der DataFrame durchsucht, so wird nur in den Spaltenberschriften gesucht.

Out[37]:
Index(['Amsterdam', 'Tokyo', 'Toronto'], dtype='object')

Out[38]:
True

# False
Out[39]:
False

# True
